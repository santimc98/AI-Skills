# Quality Standards

## Objetivo

Definir criterios mínimos de calidad que deben respetar todas las skills.

## Criterios generales

1. Claridad: el resultado debe entenderse sin explicación adicional innecesaria.
2. Trazabilidad: las conclusiones deben estar conectadas con datos, documentos o supuestos explícitos.
3. Utilidad: el entregable debe estar listo para usar o requerir pocos ajustes.
4. Precisión: no inventar datos, responsables, fechas ni decisiones.
5. Estructura: usar secciones, tablas o listas cuando mejoren la comprensión.
6. Acción: cuando proceda, incluir próximos pasos claros.

## Revisión antes de entregar

Antes de entregar una respuesta, comprobar:

- ¿Está claro el objetivo?
- ¿Se han separado hechos, inferencias y recomendaciones?
- ¿Falta algún dato crítico?
- ¿El formato sirve para el destino final?
- ¿Hay riesgos o limitaciones que deban declararse?

## Errores graves

- Inventar información.
- Ocultar incertidumbre relevante.
- Dar una recomendación sin justificarla.
- Usar una plantilla incorrecta para la tarea.
- Actuar sobre sistemas reales sin confirmación cuando pueda modificar información importante.
