# Executive Reporting Skill

## Cuándo usar esta skill

Usar esta skill cuando la tarea implique preparar informes, resúmenes, comparativas o recomendaciones para responsables de área, dirección o equipos no necesariamente técnicos.

## Objetivo

Transformar información compleja en una explicación clara, breve, accionable y orientada a toma de decisiones.

## Entradas necesarias

- Tema del informe.
- Audiencia destinataria.
- Decisión que se quiere facilitar.
- Datos, evidencias o documentos base.
- Restricciones, riesgos o alternativas.

## Procedimiento paso a paso

1. Identificar la decisión o necesidad de negocio.
2. Resumir el contexto en pocas líneas.
3. Extraer hallazgos clave.
4. Comparar opciones si aplica.
5. Indicar riesgos, costes, beneficios e impacto.
6. Formular una recomendación clara.
7. Proponer próximos pasos.
8. Evitar tecnicismos innecesarios, manteniendo las limitaciones importantes visibles.

## Criterios de calidad

- El informe debe permitir tomar una decisión.
- Las recomendaciones deben estar justificadas.
- Las limitaciones deben estar explícitas.
- Debe priorizar claridad sobre exhaustividad.
- Debe diferenciar hechos, inferencias y opinión profesional.

## Formato de salida esperado

```markdown
# Título

## Resumen ejecutivo

## Contexto

## Hallazgos clave

## Opciones analizadas

## Recomendación

## Riesgos y mitigaciones

## Próximos pasos
```

## Errores comunes a evitar

- Hacer un resumen demasiado técnico.
- No dar una recomendación clara.
- No declarar incertidumbre relevante.
- Mezclar detalles operativos con decisiones ejecutivas.
- No conectar datos con impacto de negocio.
