# SharePoint Document Management Skill

## Cuándo usar esta skill

Usar esta skill cuando la tarea implique organizar, preparar, resumir, clasificar o estructurar documentación en SharePoint, OneDrive o repositorios documentales similares.

## Objetivo

Ayudar a que la documentación empresarial quede ordenada, localizable, reutilizable y preparada para compartir con equipos internos.

## Entradas necesarias

- Tipo de documento o conjunto de documentos.
- Objetivo de la organización o transformación.
- Audiencia o equipo destinatario.
- Estructura actual, si existe.
- Criterios de clasificación, permisos o nomenclatura, si existen.

## Procedimiento paso a paso

1. Identificar el tipo de documento y su finalidad.
2. Proponer o seguir una estructura de carpetas lógica.
3. Usar nombres claros, consistentes y con fecha o versión si aplica.
4. Separar documentación final, borradores, anexos y evidencias.
5. Resumir documentos cuando sea útil para facilitar búsqueda y reutilización.
6. Preparar metadatos recomendados si el contexto lo requiere.
7. Indicar riesgos de duplicidad, desactualización o permisos.
8. Entregar una propuesta clara y accionable.

## Criterios de calidad

- La estructura debe ser fácil de navegar.
- Los nombres deben ser consistentes.
- Debe quedar claro qué documento es fuente oficial.
- Deben evitarse duplicidades.
- Debe indicarse si falta información para clasificar correctamente.

## Formato de salida esperado

```markdown
## Objetivo

## Estructura propuesta

## Nomenclatura recomendada

## Documentos principales

## Documentos auxiliares

## Riesgos o consideraciones

## Próximos pasos
```

## Errores comunes a evitar

- Crear demasiadas carpetas.
- Usar nombres ambiguos.
- Mezclar borradores con documentos finales.
- No indicar versión o fecha cuando sea importante.
- Proponer cambios de permisos sin confirmación.
